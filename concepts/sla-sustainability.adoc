= Keystone的可持续性 SLO
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
可持续性服务水平目标（SLO）的目标是，为统一（不包括 _Value_ 性能服务水平）和块优化存储或每个 AFX 群集提供性能服务水平实例（PSLI）而部署的 STaaS 产品的功耗高达最大额定功率。了解 Keystone 的可持续发展目标如何影响您的 STaaS 设置，以及如果未达到限制，如何赚取服务积分。



== 指标

* *瓦特*：存储阵列或 AFX 群集消耗的功率，并由监控工具报告。参考最大功耗限制在 link:https://hwu.netapp.com["NetApp Hardware Universe"^] 中提供。
* *太字节*：最大值：
+
** 承诺容量 + 性能服务级别的分配突发容量，或
** 有效部署容量(假设存储效率系数为 `2:1`)。




要了解有关存储效率比率的详细信息，请参见 https://docs.netapp.com/us-en/active-iq/task_analyze_storage_efficiency.html["分析容量和存储效率节省情况"^]。



=== 支持的配置

可持续性 SLO 适用于具有以下支持配置的标准 Keystone STaaS 订单：

* 仅 ONTAP AFF A 系列存储阵列或 AFX 集群。
* 仅限双节点无开关集群配置。
* 仅限非混合配置。
* 仅限非 MetroCluster 配置。
* 在所有存储阵列上全面持续启用监控工具和 AutoSupport (ASUP)。
* 测量功耗时的环境工作环境温度必须介于 25°C 和 27°C 之间。

=== 服务信用请求

要申请服务消费金，请在服务月结束后 7 天内向 NetApp Keystone 支持开具优先级 3 (P3) 支持工单，并在 excel 表格中提供所需的详细信息，如下所示：

image:sla-breach.png["Excel工作表模板"]

请注意以下服务积分请求条件：

* NetApp 确定是否发生服务级别违规。
* 您必须按照 NetApp 的要求提供其他信息，包括您的瓦数消耗数据和测量方法。
* 服务积分仅根据承诺容量计算。
* 服务积分只能应用于下个月的发票。它们不适用于过期的 Keystone 订阅。要联系 NetApp 支持并打开服务积分请求所需的支持案例，请参阅link:../concepts/support-services.html["获取Keystone帮助"]。
