= 针对特定卷类型的 Keystone 计费
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
对于某些卷配置，Keystone 计费规则有所不同。了解这些例外情况有助于在处理克隆的卷、SnapMirror 目标、LUN 或临时存储时避免意外费用。

下表总结了每种卷类型的计费规则。

[cols="1,2"]
|===
| 卷类型 | 计费规则 


| 克隆卷 | 按标准卷计费 


| 临时卷 | 未结算 


| SnapMirror 目标 | 按目标性能服务级别计费 


| LUN 数 | 按包含卷的性能服务级别计费 


| 系统卷和根卷 | 已监控但未计费 
|===


== 为克隆的卷计费

克隆的卷作为标准卷计费，其性能服务级别与从中克隆的卷相同。无论其大小或用途如何，都会对所有克隆卷进行测量和计费。



== 临时卷的计费

有时， ONTAP 会在移动卷时创建临时（ TMP ）卷。这些临时卷的使用寿命很短，不会对这些卷上的使用量进行计费。



== 为 SnapMirror 目标计费

SnapMirror 目标卷的定价（无论用于灾难恢复还是长期保留）均基于分配给目标卷的性能服务级别。数据保护无需额外付费。



== LUN 计费

对于 LUN，计费基于卷的性能服务级别。



== 系统卷和根卷

系统卷和根卷会作为Keystone服务整体监控的一部分进行监控、但不会计入或计费。这些卷上的消费免除计费。
