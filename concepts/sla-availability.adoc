= Keystone的可用性 SLO
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
可用性服务级别目标 (SLO) 针对统一和块优化存储的每个性能服务级别实例 (PSLI) 或每个 AFX 存储控制器的目标服务可用性为 99.999%。详细信息包括如何衡量可用性、包括的存储类型和性能服务级别，以及未达到目标时的服务积分。



== 指标

* *每月正常运行时间百分比* = [(当月符合条件的秒数 - 当月 PSLI 不可用服务的秒数) / 当月符合条件的秒数] x 100%
* *服务不可用*：监控工具报告的数据读取或写入操作未由 PSLI 提供服务的期间。它不包括由于计划的维护、升级或与 NetApp 商定的支持活动而导致 STaaS 服务不可用的时间，或者由于超出 NetApp 控制或责任范围的情况而导致服务不可用的时间。
* *合格秒数*：计算正常运行时间的月内秒数，不包括上述定义的任何服务不可用期。

=== 服务抵免额计算

服务抵免额通过以下公式确定：

服务抵免额=(受影响容量/承诺总容量) X容量费用X抵免百分比

其中：

* *受影响容量*：受影响的已存储容量。
* *总承诺容量*：Keystone 订单的性能服务级别的承诺容量。
* *容量费用*：该月受影响性能服务级别的费用。
* *信用百分比*：服务信用的预定百分比。


* 示例 *

以下示例显示了计算服务抵免额的方法：

. 计算每月正常运行时间以确定服务信用百分比：
+
** 30天月份中符合条件的秒数：30 (天) X 24 (小时/天) X 60 (分钟/小时) X 60 (秒/分钟)= 2、592、000秒
** 服务不可用秒数：95 秒
+
使用公式：每月正常运行时间百分比=[(2,592,000 - 95)/(2,592,000 )] X 100

+
经计算，每月正常运行时间为 99.996%，服务积分百分比为 5%。



. 计算服务抵免额：
+
|===


| * 服务级别 * | *受影响容量* | *已提交总容量* | *容量费用* | *信用百分比* 


 a| 
极高
| 10 Tib、持续95秒 | 100 Tib | 1,000 | 5% 
|===
+
使用公式：服务抵免额=(10/100) X 1000 x 0.05

+
根据计算，服务积分为 $5。





=== 服务信用请求

如果检测到违反SLA、请向NetApp Keystone支持部门开立优先级3 (P3)支持服务单。

* 需要提供以下详细信息：
+
.. Keystone订阅编号
.. 卷和存储控制器详细信息
.. 问题的站点、时间、日期和描述
.. 延迟检测的计算持续时间
.. 测量工具和方法
.. 任何其他适用文档


* 请在 Excel 表格中提供使用 NetApp Keystone 支持打开的 P3 工单的详细信息，如下所示。
+
image:sla-breach.png["Excel工作表模板"]



[NOTE]
====
* 服务信用请求应在NetApp Keystone支持确认违规后六周内发起。所有服务抵免额均应由NetApp确认和批准。
* 服务积分可能会应用于未来的发票。服务积分不适用于过期的 Keystone 订阅。要联系 NetApp 支持部门并打开服务积分请求所需的支持案例，请参阅 link:../concepts/support-services.html["获取Keystone帮助"]。
